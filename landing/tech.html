How It Works

The technical architecture behind DiscordCash.

Architecture

Zcash Shielded Pool

Entry & exit. Deposits and withdrawals hidden on-chain.

↓
Cashu Blind Mint

Server signs tokens it cannot read. Can't link deposits to withdrawals.

↓
Client-Side Wallet (Discord Activity)

Runs in your browser. Tokens stored locally. Server never sees balances.

↓
Discord

Reactions and tip counts are public. Amounts are not.

Tip Flow

1. React 💰 on a message. Bot creates a pending tip. No money moves yet.

2. Confirm in the Activity panel inside Discord. If the Activity is already open, it auto-detects the pending tip and prompts you. Your browser selects tokens, blinds the change, and submits to the server. The server escrows the tip amount without seeing it.

3. Claim. Recipient opens the Activity, sees the pending tip, clicks claim. Their browser mints fresh blind tokens.

Deposit & Withdrawal

Deposit: Open Activity → get a shielded z-address → send ZEC → mint blind tokens in your browser.

Withdrawal: Open Activity → burn tokens → ZEC sent to your z-address via shielded pool.

Anonymous Tips

/anontip @user generates a ZIP-321 payment link. You send from your own wallet. The bot only looked up the address — it's not involved in the transaction.

Privacy Summary

DataVisibility
Who tipped whom Visible Discord reaction (public by design)
Tip amounts Hidden Blinded client-side, server never sees
User balances Hidden Stored in browser localStorage
Deposit → tip → withdrawal trail Hidden Blind signatures break the link
On-chain deposits & withdrawals Hidden Zcash shielded z-addresses
Anonymous tips (/anontip) Hidden Wallet-to-wallet, bot not involved

Stack

Server: Python, aiohttp, aiosqlite, coincurve (secp256k1). Implements Cashu NUT-00 through NUT-05.

Client: Vite, @cashu/cashu-ts, @discord/embedded-app-sdk. Runs as a Discord Activity.

On-chain: zcashd for shielded transactions. Deposits via z-addresses, withdrawals via z_sendmany.

Protocol: ZIP-321 for anonymous tips. Cashu NUTs for the blind token mint.
